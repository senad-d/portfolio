# Content Style Guide

Use this guide when writing portfolio project entries.

## Tone

- Professional, concise, and evidence-based.
- Prefer measurable outcomes over generic claims.

## Project entry structure

1. Problem/Context
2. Approach/Architecture
3. Result/Impact

## Privacy and sanitization

- Do not publish secrets, internal URLs, customer-sensitive data, or private contact details.
- Generalize confidential client details when needed.
